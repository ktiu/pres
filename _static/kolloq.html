Lehrphilosophie

1/4 Transparenz

  • Erforderliche Leistungen und detaillierte Bewertungskriterien sind den Studierenden von vorneherein bekannt.
  • Alle Termine und Regelungen (zu Anwesenheit, Aufgabenbearbeitung) werden in der Vorbereitungssitzung verbindlich vereinbart.
  • Übersichtliche Lehrmaterialien beinhalten diese Informationen und stehen online durchgängig zur Verfügung.

2/4 Geschützte Räume

  • Alle können sich im Plenum respektiert und sicher fühlen: Rassistische, sexistische und andere ausgrenzende Äußerungen werden benannt und unterbunden.
  • Es gibt einen vertrauensvollen Rahmen für ehrlichen Austausch.
  • Schwierigkeiten und Momente des Scheitern werden ernst genommen und konstruktiv bearbeitet.

3/4 Praktische Übungen

  • Teilnehmende werden zu einer informierten eigenständigen Anwendung befähigt.
  • Es stehen variantenreiche Übungen zu allen behandelten Verfahren und Techniken zur Auswahl.
  • Lösungsorientiertes Denken wird auch durch kurzzeitige, strategische Überforderung der Teilnehmenden eingeübt.
  • Bei der Besprechung der Aufgaben spielen Lösungsstrategien und Austausch über weiterführende Ressourcen eine wichtige Rolle.

4/4 Kritische Reflexion

  • Quantitative Verfahren werden historisch und gesellschaftlich kontextualisiert, und ihre Limitationen werden aufgezeigt.
  • Veranstaltungen sprechen auch Teilnehmende an, die kein Interesse an der praktischen Anwendung quantitativer Verfahren haben.
  • Ergebnisse der kritischen sozialwissenschaftlichen Forschung zu Praktiken der digitalen Wissensproduktion werden vermittelt.

Gleiche Wahrscheinlichkeit der Lösungen (Multiple Choice)

equalized_sample <- function(path, n=5) {
  all <- yaml::read_yaml(path)
  category_weights <- rlist::list.group(all, a) %>%
    map(~ 1/length(.x))
  weights <- all %>%
    map(~ pluck(category_weights, as.character(.x$a)))
  sample(all, n, prob = weights)
}

Randomisierung plausibler Zahlen (Textaufgaben)

generate_numbers <- function(n, mean, sd, min, max, precision) {
  result <- rnorm(n, mean, sd) %>%
    round(precision)
  if(min(result) >= min & max(result) <= max) {
    return(result)
  } else {
    return(generate_numbers(n, mean, sd, min, max, precision))
  }
}

Abstraktion von Lösungswegen (Textaufgaben)

get_mean <- function(data = NULL, symbol = "x") {
  result <- list()
  result$formel <- sprintf("$\\bar{%s}=\\frac{\\sum\\limits_{i=1}^{n}%s_{i}}
                             {n}$",
                           symbol,
                           symbol)
  if(!is.null(data)){
    result$einsetzen <- sprintf("$\\bar{%s}=\\frac{%s}{%s}$",
                                symbol, fmt(sum(data)),
                                length(data))
    result$raw       <- round(mean(data), 2)
    result$fmt       <- fmt(result$raw)
    result$ergebnis  <- sprintf("$\\bar{%s}=%s$", symbol, result$fmt)
  }
  return(result)
}

Fazit und Ausblick

Fazit

  • Schwierige Balance zwischen Abstraktion und Flexibilität
  • Hoher (einmaliger) Aufwand für Formalisierung von Lösungswegen
  • Errungenschaften:
    • Keine Rechenfehler mehr in Skripten und auf Folien?

    • Erleichterung der Korrektur
